# TR87
**Status:** 4/6 local, 1/6 API | **Tags:** keyboard | **Baselines:** 37, 30, 39, 29, 63, 119

## Mechanic
Rule rotation puzzle. Player sees a top row (antecedents) and bottom row (consequents). Rules define input‚Üíoutput sprite mappings. Player rotates rules (ACTION1/2) and moves cursor (ACTION3/4) to make the bottom row match what the rules predict from the top row.

## Win Condition
`bsqsshqpox()` returns True when all antecedent sprites can be matched to consequent sprites using the transformation rules in `cifzvbcuwqe`. Triggers animation then `next_level()`.

## Actions
- ACTION1: rotate selected rule counterclockwise (cycles through 7 sprite variants)
- ACTION2: rotate selected rule clockwise
- ACTION3: move cursor left through rules
- ACTION4: move cursor right through rules
- Step counter: `upmkivwyrxz` decrements each action. Lose when 0.

## Solver Logic
1. Extract rules from game state: `cifzvbcuwqe` = list of (input_sprites, output_sprites) pairs
2. Read top row (antecedents `zvojhrjxxm`) and bottom row (consequents `ztgmtnnufb`)
3. Apply rules to top row ‚Üí compute expected bottom row
4. For each position in bottom row, compute how many ACTION1/2 cycles needed to rotate current sprite to target
5. Compute ACTION3/4 cursor moves between positions
6. Output minimal action sequence

## Per-Level Notes

## Key Sprite Tags
- `nxkictbbvzt` / `nxkictbbvztedxeenecwqa` ‚Äî rule sprites (7 rotational variants each, name ends in 1-7)
- `tjaqvwdgkxe` ‚Äî tree translation connectors (used in double_translation mode)

## Level Data Flags
- `alter_rules`: rules are randomized by seed ‚Äî breaks current solver
- `tree_translation`: rules chain through tree structure
- `double_translation`: rules apply twice via connector sprites
