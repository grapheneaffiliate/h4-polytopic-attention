# LP85
**Status:** 5/8 solved | **Tags:** click | **Baselines:** 33, 22, 31, 23, 33, 34, 73, 173

## Mechanic
Click buttons that apply cyclic permutations to sprite positions on a track. Win when goal sprites land next to movable container sprites.

## Win Condition
For each movable sprite `bghvgbtwcb` at (x,y), a goal sprite `odkpvwbihk` must be at (x+1, y+1). Also checks `goal-o` sprites for `movable_b` containers.

## Solver: `scripts/solve_lp85.py`
Abstract permutation BFS. Extracts button→permutation mappings from sprite positions, BFS on pure tuple state at 28K states/s.

### Critical Bug Fixed
**Dict key collision:** `target_positions_a` was a dict keyed by sprite name. Multiple movable sprites with the SAME name (`bghvgbtwcb`) caused the second target to overwrite the first. Fixed to use a list of `(name, target_pos)` tuples. This one-line fix recovered 5 levels.

## Explorer Issue
LP85 with explorer v6 (200K+ actions) exhausts GitHub Actions runner memory and crashes the process. Always use precomputed abstract solver. The explorer works standalone but kills the combined runner.

## Camera Scaling
Scale changes PER LEVEL (level 0: scale=2 offset=(0,13), level 1: scale=1 offset=(11,11)). Raw display coordinates from the abstract solver work on API because the API creates games with the same camera.
