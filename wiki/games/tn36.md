# TN36
**Status:** 5/7 solved | **Tags:** click | **Baselines:** 23, 22, 26, 37, 25, 56, 61

## Mechanic
Programming puzzle. Toggle binary buttons on a tablet to create an opcode program, then click the color selector to execute. Block moves according to opcodes. Win when block matches target in position, rotation, scale, AND color.

## Win Condition
`yxabhsirzl`: `ravxreuqho.x == ddzsdagbti.x AND .y AND .scale AND .rotation AND .dtxpbtpcbh (color)`

## Key Discovery: Deepcopy Closure Bug
Lambda closures in `dfguzecnsr` capture original `self`. After deepcopy, opcodes execute on the WRONG block. Must use reset+replay or direct `perform_action` for testing.

## Button-to-Action Mapping
- Top buttons (y~42): action indices 0..n_rows-1, toggle bit 0 of each row
- Bottom buttons (y~44): action indices n_rows+1..2*n_rows, toggle bit 1
- Color selector: action at (34,51) for level 0, changes per level
- Selection buttons (for multi-program levels): switch between program slots

## Multi-Program Levels (3+)
Levels 3+ have selection buttons and level data: Programs, Positions, Rotations, Scales, Reset flags. The right block has the editable tablet and target. Left block is read-only. Win check is ONLY on right block.

## Wall Collision
Walls are `qoyvlwczjf` objects. Block CAN pass through gap between walls (4px gap for 4px block). Wall at (33,16) size 16×4 + wall at (53,16) size 8×4 = gap at x=[49,53).
