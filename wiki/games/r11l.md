# R11L
**Status:** 0/6 unsolved | **Tags:** click | **Baselines:** 7, 28, 30, 20, 37, 45

## Mechanic
Click-drag puzzle. Select legs of creatures, move them to target positions. 60 action limit, 5 collision max.

## Win Condition
All creature bodies collide with their target templates (kzeze sprites). Body pixel colors must match target.

## Why Unsolved
- 3095 unique click positions that change the frame (step=1 grid scan)
- Only 2 sys_click targets (not enough)
- Level 0 baseline=7 but 3095 branching factor makes BFS impossible
- Unique-effect dedup gives ~3095 effects — almost no dedup

## Key Sprites
- `bdkaz` = body sprites, `bdkazLeg` = leg sprites
- `kzeze` = target template sprites
- `bvzgd-*` = obstacle sprites
- `xigcb-*` = paint/collectible sprites

## Click Handling
256 possible positions (64×64, step=4). Click on leg → select it. Click on position → move selected leg there. Collision with obstacles increments penalty counter.

## What's Needed
Source code analysis to understand which SPECIFIC positions matter. The solver needs to:
1. Identify which legs belong to which creatures
2. Compute target positions from kzeze templates
3. Plan click sequences: select leg, click target position
4. Avoid obstacles (5 collision limit)

The solution is probably: for each creature, select its leg and click the target position. With baseline=7, level 0 might need just 3-4 select+move pairs.
